# Spec To Phased Roadmap

Use `SKILL.md` as the source of truth.

When the user asks for a roadmap from specification documents, create or update:

- `docs/roadmap/roadmap_desarrollo_<project>.md`
- `docs/roadmap/<project-slug>-development-progress.md`

Follow the references in this order:

1. `references/extraction-rules.md`
2. `references/roadmap-format.md`
3. `references/phase-rules.md`
4. `references/progress-format.md`
5. `references/update-rules.md`

Keep one roadmap and one progress document. Do not create per-phase documents unless the user explicitly asks.
